## 1. Project Overview

**Project Name:** musicme.ru
**Project Type:** SaaS сервис создания персонализированных музыкальных треков
**Target Audience:** Люди, желающие создать уникальный музыкальный подарок для близких или для себя.
**Core Value Proposition:** "Уникальная песня-подарок на любой бюджет. Платите только если результат вам понравится"

## 2. Problem Statement

### Проблемы пользователей:
- Хотят дарить эмоциональные, запоминающиеся подарки (свадьба, день рождения, годовщина)
- Не имеют музыкальных навыков для создания песен
- Стандартные подарки кажутся банальными и безликими
- Не готовы платить за результат, который может не понравиться
- Имеют разный бюджет на подарки

### Рыночная возможность:
Рынок персонализированных подарков в России растет, а технологии AI делают создание музыки доступным для не-музыкантов. При этом существует запрос на разные уровни персонализации и цены.

## 3. Solution & Pricing Tiers

Платформа для создания кастомных музыкальных треков с уникальной бизнес-моделью и гибкой тарифной системой.

### Тарифная система:

| Тариф | "Базовый" | "Продвинутый" | "Премиум" |
|-------|------------|---------------|-----------|
| **Цена** | 2,900 руб. | 4,900 руб. | 9,900 руб. |
| **Процесс** | Форма на сайте | Форма + детальная анкета | Форма + видео-интервью 30 мин |
| **Срок** | 24-48 часов | 2-3 дня | 3-5 дней |
| **Персонализация** | Общие параметры | Глубокая текстовая проработка | Максимальная, с уточнением деталей |
| **Правки** | 1 раунд | 2 раунда | Неограниченно (в рамках адекватного) |
| **Уровень контроля** | AI + валидация | AI + активное участие продюсера | Персональный продюсер |
| **Видеосвязь** | - | - | Яндекс.Телемост |

### Ключевые особенности:
- **Гибкая тарифная система** — выбор уровня персонализации под бюджет и задачу
- **Модель "риск на нас"** — оплата только после одобрения preview-версии (для всех тарифов)
- **Preview система** — прослушивание 60 секунд трека перед оплатой
- **Интеграция с Яндекс.Телемост** — для тарифа "Премиум"

## 4. Target Audience

### Основные сегменты:
1. **Студенты/молодежь** (18-25 лет) → Тариф "Базовый" (подарки друзьям)
2. **Молодые пары** (25-35 лет) → Тариф "Продвинутый" (годовщины, предложения)
3. **Состоявшиеся профессионалы** (30-50 лет) → Тариф "Премиум" (важные события, корпоративы)
4. **Коллеги по работе** → "Базовый"/"Продвинутый" (корпоративные поздравления)

## 5. User Stories & Acceptance Criteria

### Epic: Выбор тарифа

**US-000: Как новый пользователь, я хочу сравнить тарифы и выбрать подходящий**
- [ ] AC1: На главной странице представлены 3 тарифа в виде сравнения
- [ ] AC2: Для каждого тарифа указаны: цена, срок, что входит, пример результата
- [ ] AC3: Кнопка "Выбрать" ведет на соответствующую форму заказа
- [ ] AC4: Есть рекомендация "Самый популярный" для тарифа "Продвинутый"

### Epic: Заказ песни

**US-001: Как пользователь, я хочу заказать песню по тарифу "Базовый"**
- [ ] AC1: Форма с полями: "Повод", "Для кого", "Жанр", "Ключевые слова/фразы", "Срок"
- [ ] AC2: OAuth авторизация
- [ ] AC3: После отправки формы - уведомление в Telegram о принятии заказа

**US-002: Как пользователь, я хочу заказать песню по тарифу "Продвинутый"**
- [ ] AC1: Форма "Базового" тарифа + расширенная анкета (20-30 вопросов о событии, предпочтениях, эмоциях)
- [ ] AC2: Примеры вопросов: "Опишите 3 самых важных качества человека", "Какая музыка нравится?", "Какие воспоминания связаны с событием?"

**US-003: Как пользователь, я хочу заказать песню по тарифу "Премиум"**
- [ ] AC1: Форма "Продвинутого" тарифа + выбор даты для видео-интервью
- [ ] AC2: Интеграция с Яндекс.Телемост для бронирования видеовстречи
- [ ] AC3: Автоматическая отправка ссылки на Яндекс.Телемост после бронирования

**US-004: Как пользователь, я хочу получить уведомление о готовности трека**
- [ ] AC1: Telegram-уведомление со ссылкой на страницу прослушивания
- [ ] AC2: На странице доступно 60 секунд preview трека
- [ ] AC3: Кнопка "Оплатить полную версию" с соответствующей ценой тарифа
- [ ] AC4: После оплаты доступно скачивание полной версии + возможность запросить правки

### Epic: Администрирование

**US-101: Как администратор, я хочу видеть новые заказы с указанием тарифа**
- [ ] AC1: Telegram-уведомление о новом заказе с указанием тарифа и сроков
- [ ] AC2: Dashboard со списком заказов, фильтрация по тарифу и статусу
- [ ] AC3: Цветовая индикация срочности по тарифам

**US-102: Как продюсер, я хочу получать задания в соответствии с тарифом**
- [ ] AC1: Для "Базового": форма + генерация через Suno API + быстрая проверка
- [ ] AC2: Для "Продвинутого": форма + анкета + ручная работа с промптами
- [ ] AC3: Для "Премиум": полный доступ к материалам + запись интервью через Яндекс.Телемост

## 6. Functional Requirements

### Система заказов:
- Создание заказа с разным набором полей в зависимости от тарифа
- OAuth авторизация
- Система статусов заказа с учетом сроков по тарифам
- Хранение истории заказов

### Генерация контента:
- Интеграция с Suno AI API
- Различные workflow в зависимости от тарифа
- Создание preview (первые 60 секунд)
- Watermarking для защиты контента
- Хранение аудио файлов

### Платежная система:
- Интеграция с ЮKassa
- Разные цены в зависимости от тарифа
- Webhook для обработки платежей
- Контроль доступа к контенту после оплаты

### Уведомления:
- Telegram уведомления для пользователей (разные тексты по тарифам)
- Telegram уведомления для администратора и продюсеров
- Напоминания о приближении дедлайнов

### Видеосвязь:
- Интеграция с Яндекс.Телемост API
- Автоматическое создание видеоконференций
- Отправка приглашений пользователям

## 7. Non-Functional Requirements

### Производительность:
- Время загрузки страницы: < 3 секунд
- Время обработки платежа: < 10 секунд
- Поддержка 100+ одновременных пользователей

### Безопасность:
- HTTPS для всех endpoints
- Валидация всех входных данных
- Rate limiting на API (особенно для генерации)
- Защита от XSS и SQL-инъекций

### Надежность:
- Uptime: 99.5%
- Резервное копирование данных ежедневно
- Мониторинг ошибок и производительности

## 8. Technical Architecture

### Frontend:
- React + TypeScript
- Tailwind CSS для стилей
- Axios для API запросов

### Backend:
- FastAPI + Python 3.11
- PostgreSQL для данных
- JWT аутентификация
- Celery для фоновых задач (генерация музыки)

### Внешние интеграции:
- Suno AI API - генерация музыки
- ЮKassa - платежи
- Telegram Bot API - уведомления
- OAuth провайдеры (VK, Yandex, Google)
- Яндекс.Телемост API - видеоконференции

## 11. Risks & Mitigations

### Технические риски:
- **Suno API limitations** → Приоритизация запросов по тарифам, очередь заданий
- **Сложность управления разными процессами** → Четкие чеклисты для каждого тарифа
- **Проблемы с Яндекс.Телемост** → Резервный вариант с телефонным звонком

### Бизнес-риски:
- **Каннибализация дорогих тарифов** → Четкая коммуникация ценности, ограничение функционала младших тарифов
- **Низкая конверсия на дорогих тарифах** → Усиление социального доказательства для "Премиум"
- **Высокая нагрузка на поддержку** → Чат-бот для частых вопросов, база знаний

### Legal риски:
- **Авторские права** → Четкие условия использования для всех тарифов
- **Обработка персональных данных** → Политика конфиденциальности
- **Запись видео-интервью** → Получение согласия пользователя на запись и обработку
