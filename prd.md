# PRD: musicme.ru

## 1. Project Overview

**Project Name:** musicme.ru  
**Project Type:** SaaS сервис создания персонализированных музыкальных треков  
**Target Audience:** Люди, желающие создать уникальный музыкальный подарок для близких или для себя.
**Core Value Proposition:** "Уникальная песня-подарок. Платите только если результат вам понравится"

## 2. Problem Statement

### Проблемы пользователей:
- Хотят дарить эмоциональные, запоминающиеся подарки (свадьба, день рождения, годовщина)
- Не имеют музыкальных навыков для создания песен
- Стандартные подарки кажутся банальными и безликими
- Не готовы платить за результат, который может не понравиться

### Рыночная возможность:
Рынок персонализированных подарков в России растет, а технологии AI делают создание музыки доступным для не-музыкантов.

## 3. Solution

Платформа для создания кастомных музыкальных треков с уникальной бизнес-моделью:

### Ключевые особенности:
- **Модель "риск на нас"** - оплата только после одобрения результата
- **Персональный подход** - интервью с каждым клиентом для глубокого понимания задачи
- **AI + человеческий контроль** - генерация через Suno AI с кураторством
- **Preview система** - прослушивание 1 минуты трека перед оплатой

### Ценностное предложение:
"Мы создаем персональные песни-поздравления для тех, кто ценит искренние эмоции. В отличие от стандартных подарков, вы платите только когда результат полностью вас устраивает."

## 4. Target Audience

### Основные сегменты:
1. **Молодые пары** (25-35 лет) - песни на годовщины, предложения руки
2. **Друзья и родственники** (20-50 лет) - подарки на дни рождения
3. **Коллеги по работе** - корпоративные поздравления

### География: Россия и СНГ
### Язык: Русский

## 5. User Stories & Acceptance Criteria

### Epic: Заказ песни

**US-001: Как новый пользователь, я хочу ознакомиться с услугами на лендинге**
- [ ] AC1: На главной странице отображается УТП "Платите только если понравится результат"
- [ ] AC2: Показаны 6-8 готовых работ с аудио и историями
- [ ] AC3: Есть раздел "Как это работает" с пошаговой инструкцией
- [ ] AC4: Кнопка "Заказать песню" ведет к форме заказа

**US-002: Как пользователь, я хочу создать заказ на песню**
- [ ] AC1: Форма с обязательными полями: "Повод", "Для кого", "Жанр", "Особые пожелания" и требуемые сроки исполнения.
- [ ] AC2: OAuth авторизация через VK/Yandex/Google
- [ ] AC3: После отправки формы показывается ссылка для записи на интервью
- [ ] AC4: Пользователь получает в telegram подтверждение заказа, бот сообщает текущий статус.

**US-003: Как пользователь, я хочу получить уведомление о готовности трека**
- [ ] AC1: telegram - уведомление со ссылкой на страницу прослушивания
- [ ] AC2: На странице доступно 60 секунд preview трека
- [ ] AC3: Кнопка "Оплатить полную версию" (9900 руб)
- [ ] AC4: После оплаты доступно скачивание полной версии

### Epic: Администрирование

**US-101: Как администратор, я хочу видеть новые заказы**
- [ ] AC1: Telegram-уведомление о новом заказе
- [ ] AC2: Dashboard со списком заказов и статусами
- [ ] AC3: Фильтрация заказов по статусу и дате

**US-102: Как администратор, я хочу управлять генерацией треков**
- [ ] AC1: Интеграция с Suno API для генерации музыки или загрузка вручную
- [ ] AC2: Возможность назначить на каждый заказ конкретного испольнителя-продюсера
- [ ] AC3: Возможность загрузить несколько вариантов трека к одному заказу
- [ ] AC4: Система статусов для отслеживания прогресса

**US-201: Как продюсер, я хочу получать задания и управлять статусами**
- [ ] AC1: Telegram-уведомление о новом заказе в работе
- [ ] AC2: Возможность загрузить несколько вариантов трека к одному заказу
- [ ] AC3: Система статусов для управления прогрессом

## 6. Functional Requirements

### Система заказов:
- Создание заказа с детальным ТЗ
- OAuth авторизация
- Система статусов заказа
- Хранение истории заказов

### Генерация контента:
- Интеграция с Suno AI API
- Создание preview (первые 60 секунд)
- Watermarking для защиты контента
- Хранение аудио файлов

### Платежная система:
- Интеграция с ЮKassa
- Webhook для обработки платежей
- Контроль доступа к контенту после оплаты

### Уведомления:
- Telegram уведомления для пользователей
- Telegram уведомления для администратора
- Telegram уведомления для продюсеров
- Статусные уведомления в Telegramm

## 7. Non-Functional Requirements

### Производительность:
- Время загрузки страницы: < 3 секунд
- Время обработки платежа: < 10 секунд
- Поддержка 100+ одновременных пользователей

### Безопасность:
- HTTPS для всех endpoints
- Валидация всех входных данных
- Rate limiting на API
- Защита от XSS и SQL-инъекций

### Надежность:
- Uptime: 99.5%
- Резервное копирование данных ежедневно
- Мониторинг ошибок и производительности

## 8. Technical Architecture

### Frontend:
- React + TypeScript
- Tailwind CSS для стилей
- Axios для API запросов

### Backend:
- FastAPI + Python 3.11
- PostgreSQL для данных
- JWT аутентификация

### Внешние интеграции:
- Suno AI API - генерация музыки
- ЮKassa - платежи
- Telegram Bot API - уведомления
- OAuth провайдеры (VK, Yandex, Google)

## 9. Business Metrics & Success Criteria

### Key Performance Indicators:
- **Конверсия из preview в оплату:** > 30%
- **Среднее время выполнения заказа:** < 72 часов
- **Customer Satisfaction Score:** > 4.5/5
- **Повторные заказы:** > 15%

### Цели на первые 3 месяца:

- **Стоимость заказа:** 9,900 рублей

## 11. Risks & Mitigations

### Технические риски:
- **Suno API limitations** → Реализовать очередь заданий и fallback варианты
- **Высокая нагрузка** → Кэширование и горизонтальное масштабирование
- **Проблемы с платежами** → Retry механизм + ручная проверка

### Бизнес-риски:
- **Низкая конверсия** → A/B тестирование УТП и ценовой политики
- **Высокая нагрузка на поддержку** → Чат-бот для частых вопросов
- **Конкуренция** → Упор на уникальность модели "оплата после результата"

### Legal риски:
- **Авторские права** → Четкие условия использования
- **Правила обработки данных** → Политика конфиденциальности
